# 面试总结与简历编写指南：Related Work Agent (RWA)

### 方案一：详细版（推荐，适合作为主要项目展示）

**项目名称**：Related Work Agent (多智能体学术文献写作系统)
**项目角色**：核心开发者 / 独立开发者
**项目描述**：设计并实现了一个基于大语言模型（LLM）的多智能体自动化工作流，旨在通过论文分析与学术级检索，自动生成高质量、期刊自适应且可直接编译的论文“相关工作 (Related Work)”章节，极大地提升了学术科研人员的写作效率。
**核心工作与成果**：
*   **多模型与本地化架构集成**：设计了灵活的模型底座接口，全面对接 OpenAI、Claude、DeepSeek、阿里云通义千问等云端 API，同时支持通过 Ollama 部署本地开源模型，满足不同场景下的数据隐私隐私和跨网络运行需求。
*   **构建三阶学术级 RAG 数据管线**：针对学术论文垂直领域，结合 `allenai/specter2_base` 进行特征向量召回，引入 `ColBERTv2` 实施 Late Interaction 的词汇级精确匹配，并采用 `Cross-Encoder` 完成顶层重排序，大幅度提升了高维学术文档检索的命中率与相关性。
*   **研发期刊风格自适应引擎（Style Emulation）**：开发结构化分析模块，自动提取目标期刊的写作范式（如篇幅长度、段落逻辑、引用密度、过渡词模式），使生成的长文本精准贴合目标期刊风格。
*   **实现 LLM-as-a-Judge 反思迭代机制**：设计并落地了多轮自我评估（Self-Reflection）工作流。引入独立的“评分智能体”，从准确性、对比度、创新点覆盖度等 7 个维度对初稿进行打分并输出修改建议，闭环驱动下一轮内容重写，显著提升最终输出质量。
*   **LaTeX 原生工程化打通**：实现高度鲁棒的 LaTeX 格式生成系统，自动处理复杂特殊符号（如 `%`、`&`）的转义，并配套生成标准规范的 `.bib` 参考文献文件，实现科研写作“一键导入编译”的最终闭环。

### 方案二：精简版（适合简历空间有限，或者作为次要项目展示）

**项目名称**：Related Work Agent (多智能体论文辅助写作系统) 
**核心技术**：Python, LLM, RAG, ColBERT, Cross-Encoder, Agentic Workflow
*   **系统架构**：基于 Python 开发大模型智能体工作流，高度解耦底层 LLM（兼容 OpenAI/DeepSeek/本地 Ollama 等），系统性解决学术综述撰写过程中的信息收集与格式排版痛点。
*   **多阶段 RAG 检索**：创新性地设计了“稠密向量检索 (Specter2) + 细粒度词汇匹配 (ColBERTv2) + 语义重排 (Cross-Encoder)”的三阶学术 RAG 链路，有效解决了传统 RAG 在长文本学术专业词汇上召回率低的问题。
*   **智能体自反思闭环**：结合 Prompt 工程设计“LLM-as-a-Judge”评估机制循环，让智能体从 7 大维度自主评估草稿并迭代优化，同时引入前置的期刊风格特征提取（Style Emulation），保证生成内容的高质量与定制化水平。
*   **工程交付**：输出流接入 LaTeX 编译生态，自动修复特殊字符转义并补全 `.bib` 文件，为终端用户提供开箱即用、一键编译的学术级文档。

## 二、 面试深度问答复盘（重点 Q&A）

### Q1：项目中具体是如何使用 ColBERTv2 的？解决什么问题？
**核心解答**：
项目中**确实完整实现并使用了 ColBERTv2**，在架构设计上把它作为一个**可选的高精度检索增强模块（Stage 2）**。
*   **面临的痛点**：在检索方案初期使用了普通的 Dense 稠密向量（Specter2 模型），它在捕捉论文核心语义上表现不错，但在遇到具体的算法名称、专有名词缩写等**细粒度词汇匹配**时容易出现遗漏。
*   **解决方案**：通过引入 **ColBERTv2** 作为第二阶段检索（Late Interaction）。它保留了每个 Token 的表征，做到了词汇级的精确匹配。为了兼顾两者的优势，项目代码中通过**倒数排名融合 (RRF)** 算法将 Dense 的结果与 ColBERT 的召回结果进行了合并，最后交由 Cross-Encoder 来做精排。

### Q2：Specter2 稠密检索和 ColBERTv2 是如何结合的？是串行的过滤关系吗？
**核心解答**：
**不是单纯的串行过滤，而是“双路并发召回 + RRF 倒数排名融合”架构！**
具体执行逻辑如下：
1. **第一路召回 (Stage 1: Dense Retrieval / Specter2)**：基于摘要、贡献点等构造 Query，使用 Specter2 大模型向量化并去 Chroma 数据库搜出第一批高度相关的候选片段。
2. **第二路召回 (Stage 2: Late Interaction / ColBERTv2)**：同时提取核心 Query，去独立的 ColBERTv2 索引重新查一次。由于其支持 Token 级精确匹配，能找到大量被语义检索遗漏的核心专业词组文献。
3. **倒数排名融合合并 (RRF, Reciprocal Rank Fusion)**：设计并调用了 `_reciprocal_rank_fusion` 方法，综合一篇文献在两路并发召回中的**排名位置进行重新打分**。由此做到了在“深层语义”和“精准词汇”上的优势互补。
4. **终局精排 (Stage 3: Cross-Encoder Reranking)**：拿到上面两路融合后打分最高的**前 50 个**候选片段，给到 Cross-Encoder 交叉注意力模型进行严苛的截断精排，最终喂给大模型用于写作。

> **🌟 面试金句提取（建议背诵理解）**：
> “为了打破传统 Dense 向量检索对长尾学术专有名词召回率低的瓶颈，我设计了**多路召回与重排架构**。架构采用 **Specter2 稠密向量**与 **ColBERTv2 细粒度 Token 交互**进行双路并列召回，接着使用 **RRF（倒数排名融合）算法**对两路候选集进行科学混排，最终截断前 50 名传递给 **Cross-Encoder 交叉注意力引擎**做极限精排，在保障耗时的前提下将文献检索相关度提升到了学术级要求。”
